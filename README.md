# PixelMotion

웹 기반 도트화 + 픽셀 보정 도구. 8방향 캐릭터 스프라이트 워크플로우 용도로 만들어졌습니다.

## 기능

- **도트화**: 격자 사이즈 (48 / 64 / 128 / 256), 세분화 (1×1 ~ 4×4), 다운샘플 모드 3종 (평균 / 우세 / 디테일), 색 양자화 (8 / 16 / 32 / 64), 채도 부스트, 외곽선 (도트 / 격자 단위)
- **배경 제거**: rembg 기반 (기본 / 정밀 모드 = isnet + alpha matting)
- **영역 보정**: 사각형 / 올가미 영역 선택 → 빈 격자를 평균 색 또는 BFS 그라데이션으로 채움
- **픽셀 편집**: 스포이드 / 브러시 (큰 격자 단위 / 1×1) / 지우개
- **색 픽커**: HSV 인라인 픽커 (Hue 슬라이더 + SV 정사각), 26색 기본 팔레트, 도트화 결과 자동 팔레트 추출, HEX 입력
- **줌 / 팬**: Aseprite 스타일 화면 조작
- **변환**: 좌우 반전 (8방향 작업 시 반대 방향 자동 생성)
- **일괄 처리**: 여러 이미지를 같은 설정으로 도트화 → ZIP 다운로드 (8방향 일관성 처리의 핵심)
- **Undo**: Ctrl+Z (배경 제거 / 도트화 / 픽셀 편집 모두)

## 실행

### 의존성 설치
```bash
pip install pillow rembg onnxruntime numpy
```

### 서버 시작
```bash
python server.py
```

### 브라우저 접속
<http://127.0.0.1:8765/>

배경 제거 첫 호출 시 rembg 모델이 자동 다운로드됩니다 (~170MB, 한 번만).

| 키 | 동작 |
|----|------|
| `Ctrl+Z` | 되돌리기 |

## 추천 워크플로우 — 8방향 캐릭터

1. reference 이미지 4~8장 준비 (직접 촬영 / 3D 렌더 / AI 생성 등)
2. 한 장으로 PixelMotion에서 격자 / 모드 / 색 조합 결정
3. 일괄 처리로 모두 같은 설정 도트화 → ZIP
4. 좌우 반전으로 부족한 방향 채움 (서↔동, 북서↔북동 등)
5. [Aseprite](https://www.aseprite.org/) 등에서 프레임 정렬 + 어니언 스킨 + 미세 보정 + 스프라이트 시트 export
6. Unity / Godot 등 게임 엔진 임포트

## 한계

자동 도트화는 본질적인 한계가 있습니다:
- 격자가 작을수록 작은 디테일(머리카락 가닥, 눈 디테일 등) 손실은 수학적으로 불가피
- 배경 제거(rembg)는 픽셀 단위 정밀도가 안 됨 — 깔끔한 결과를 원하면 [Photopea](https://www.photopea.com/) 같은 외부 툴에서 누끼 후 import 권장
- "100% 게임 스프라이트 룩"은 자동만으로 안 나옴 — Aseprite 등에서 수작업 보정 필수

PixelMotion은 그 수작업 비율을 최대한 줄이는 도구입니다.
